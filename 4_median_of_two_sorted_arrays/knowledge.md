# Python bisect モジュール概要

Pythonの標準ライブラリである `bisect` モジュールは、ソート（昇順）済みのリストに対して、**効率的に要素の挿入箇所（インデックス）を二分探索で検索できる** ツールです。
線形探索の場合は `O(N)` かかるのに対し、二分探索で `O(log N)` の計算量で処理できるため、パフォーマンスが非常に高くなります。

## 主要な関数

### 1. `bisect.bisect_left(a, x, lo=0, hi=len(a))`
ソート済みのリスト `a` に要素 `x` を挿入するとき、**リストがソートされた状態を保つような挿入箇所のインデックス** を返します。
同じ値 `x` がリストにすでに存在している場合は、**一番左（最初）の `x` の位置（手前）** を挿入箇所として返します。

```python
import bisect

arr = [1, 3, 5, 5, 7]
# 5はすでに存在する。bisect_leftは既存の最初の要素(インデックス2)の位置を返す
idx = bisect.bisect_left(arr, 5) 
# idx は 2
```

### 2. `bisect.bisect_right(a, x, lo=0, hi=len(a))` または `bisect.bisect(...)`
ソート済みのリスト `a` に要素 `x` を挿入するとき、**リストがソートされた状態を保つような挿入箇所のインデックス** を返します。
`bisect_left`との違いは、同じ値 `x` がリストにすでに存在している場合、**一番右（最後）の `x` の次の位置（後ろ）** を挿入箇所として返す点です。

（※単なる `bisect.bisect` は通常 `bisect_right` のエイリアスです。）

```python
import bisect

arr = [1, 3, 5, 5, 7]
# 5はすでに存在する。bisect_rightは既存の最後の要素の次の位置(インデックス4)を返す
idx = bisect.bisect_right(arr, 5) 
# idx は 4
```

### 3. `bisect.insort_left(a, x, lo=0, hi=len(a))` 
インデックスを探すだけでなく、実際に `x` をリスト `a` へ `bisect_left` のインデックスに挿入（`insert()`に相当）してくれます。

### 4. `bisect.insort_right(a, x, lo=0, hi=len(a))` または `bisect.insort(...)`
同様に、インデックスを探して `x` をリスト `a` へ `bisect_right` の位置に挿入してくれます。

## まとめ

* 特定の値が配列のどの位置に入るかを `O(log N)` で調べたい場合は `bisect_left` か `bisect_right`。
* 値が存在した場合に「前に」（＝左）入れたいのか、「後ろに」（＝右）入れたいのかで `left`/`right` を使い分ける。
* リストを探索するだけなら `bisect_left` や `bisect_right`。挿入も同時に行う場合は `insort_left` や `insort_right` を使う。

# 4. Median of Two Sorted Arrays 攻略方針まとめ

**「`O(log(m+n))` の時間計算量で、2つのソート済み配列から中央値を求める」** という Hard 問題へのアプローチ手順。

## 1. 方針のベース: 「二分探索」
`O(m+n)` の結合＆ソートは使えないため、探索に `log` が付く「二分探索 (Binary Search)」が必須。
ソート済みの2つの配列を合体させる代わりに、**「片方の配列のどこで分割するか」** を二分探索で探す。

## 2. 「正しい分割」の定義
2つの配列（短い方を `A`、長い方を `B` とする）を仮想的に結合して、全体の「左半分」と「右半分」に分割する。
全体の左半分の要素数（奇数の場合は左の方が1個多くなる）を `$K = (m + n + 1) // 2$` とする。
`A` の分割位置を `i` とすると、`B` から左半分に選ぶ要素数は自動的に `j = K - i` と決まる。

分割における境界の値：
* `L1`: `A` の左半分の最大値 (`A[i-1]`)
* `R1`: `A` の右半分の最小値 (`A[i]`)
* `L2`: `B` の左半分の最大値 (`B[j-1]`)
* `R2`: `B` の右半分の最小値 (`B[j]`)

**「正しい分割」の条件:**
全体の左半分の全ての要素が、右半分の全ての要素以下になること。
`A` と `B` は元々ソート済みなので `L1 <= R1` と `L2 <= R2` は常に成り立つ。
したがって、**`L1 <= R2` かつ `L2 <= R1`** が成り立てば、全体の大小関係が保証され、「正しい分割」と判定できる。

## 3. 条件を満たさない時の探索方向
* **`L1 > R2` の場合:**
  `A` から左半分に取りすぎている (`L1` が出しゃばっている) 状態。
  → `A` の分割位置 `i` を**減らす（左へ探索範囲を狭める）**: `right = i - 1`
* **`L2 > R1` の場合:**
  `B` から左半分に取りすぎている (`A` からの要素が足りない) 状態。
  → `A` の分割位置 `i` を**増やす（右へ探索範囲を広げる）**: `left = i + 1`

## 4. エッジケース（配列境界エラー）の対処
`i` や `j` が `0` （左端）の時、または `m` や `n` （右端）の時、配列の境界外へアクセスしようとすると `IndexError` が発生する。
これを防ぐため、仮の「無限大」要素を置く。

* 左側に要素がない場合 (`i == 0` や `j == 0`)： `L1` や `L2` に **`float('-inf')` (マイナス無限大)** を代入する。
* 右側に要素がない場合 (`i == len(A)` や `j == len(B)`)： `R1` や `R2` に **`float('inf')` (プラス無限大)** を代入する。

こうすることで、以降の大小比較や、最終的な `max` や `min` の計算において、存在しない要素が選ばれることを確実に防げる。

## 5. 中央値の計算
正しい分割が見つかってループを抜けた後、全体の要素数 `m + n` に応じて中央値を計算する。

* **奇数の場合:** 左半分の最大値がそのまま中央値になる。
  → `max(L1, L2)`
* **偶数の場合:** 左半分の最大値と、右半分の最小値の平均が中央値になる。
  → `(max(L1, L2) + min(R1, R2)) / 2`
